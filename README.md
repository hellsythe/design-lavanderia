# Github Dashboard Design System

A production-extracted Claude Design design-system package for operational web dashboards. All tokens, component patterns, copy conventions, and interaction rules are derived from **LavanderPro** — a live industrial laundry management platform.

---

## Product Overview

**LavanderPro** is a web application for commercial and industrial laundry business owners and administrators. It provides an operational command centre for managing laundry orders, revenue, and customer relationships in real time. The application is built in vanilla HTML/CSS/JS with no framework dependencies, and is used primarily on desktop web with responsive adaptations for mobile and tablet.

**Primary surfaces:** The application ships as a single primary surface — a responsive admin dashboard. The layout consists of a fixed sidebar (256px) for navigation, a sticky topbar (56px) for page context and actions, and a scrollable main content area divided into three horizontal regions: a KPI metric strip at top, a main two-column grid (orders table + revenue/cycle widgets), and a bottom two-column grid (history table + top clients).

**Core capabilities:**

- **Active order tracking** — A real-time table of in-flight laundry orders filterable by status (en proceso / listo / pendiente / entregado). Each row is clickable and opens a modal with full order detail. The status filter uses a tab-style pill group with smooth active transitions.
- **Revenue monitoring** — A bar chart showing revenue with a period switcher (day/week/month). The current period bar is highlighted in the brand accent teal. A summary total in 30px display numerics sits above the chart.
- **Cycle time analytics** — A 44px hero numeric showing average cycle time in minutes, with a progress track against a target, and a 2×2 grid of min/max/phase stats. The progress fill uses the accent teal.
- **Order history** — A searchable, live-filtered table of completed orders. The search input uses a focus ring with a teal glow. Empty results show a labelled SVG icon + plain-text statement.
- **Top clients** — A ranked list of frequent customers with initials avatars, last-order dates, and accent-colored order counts.

**Language:** Spanish (es-MX). Copy conventions include ALL-CAPS labels, tabular-numeral data columns, and concise delta copy (`+12% vs. ayer`, `+3 vs. ayer`).

**Source project:** Github Dashboard (`c3b8a464-8510-4a9f-8f01-7d1b4aa05745`). Critique score: **5/5** — clarity 5, hierarchy 5, typography 5, motion 4, brand 5.

## Reuse Workflow

To apply this design system to a new project:

1. **Load the font and tokens** — Add Google Fonts (Public Sans 400–700) and link `colors_and_type.css` before your first `<style>` block.
2. **Review the visual language** — Open `preview/index.html` and work through each card in order.
3. **Inspect the applied surface** — Open `ui_kits/app/dashboard.html` to see all components in a real dashboard context.
4. **Copy components** — Lift HTML + class rules from `preview/components.html` or `ui_kits/app/dashboard.html`.
5. **Bind tokens** — All colors, spacing, radius, shadow, and typography are custom properties in `colors_and_type.css`. Use `var(--token-name)` throughout.
6. **Read the anti-patterns** — `DESIGN.md` section 9 lists 12 visual and interaction choices this system explicitly avoids.
7. **Check provenance** — `context/provenance.md` documents token fidelity and what was extracted vs. what was inferred.

```html
<!-- Step 1: Load font + tokens -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link href="https://fonts.googleapis.com/css2?family=Public+Sans:wght@400;500;600;700&display=swap" rel="stylesheet">
<link rel="stylesheet" href="path/to/colors_and_type.css">
```

Key tokens for a new project:
```css
--bg: #F0F4F8;      /* page canvas — cool blue-gray, never warm beige */
--surface: #FFFFFF; /* card / panel background */
--fg: #1A2332;      /* primary text — deep navy */
--muted: #64748B;   /* secondary text, icons */
--border: #E1E8F0;  /* hairline borders */
--accent: #0F766E;  /* teal — use at ≤6 anchor points per screen */
--radius: 10px;     /* default card border-radius */
```
